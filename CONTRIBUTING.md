# Contributing Guide

Thanks for your interest in contributing to Git Shortcuts Toolkit.

## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch:
   ```bash
   git checkout -b feature/my-change
    ```

## Guidelines

* Keep commands minimal and predictable
* Do not override built-in `git` commands
* Maintain cross-shell compatibility (bash, zsh, fish)
* Update documentation when adding new commands

## Testing

Before submitting a PR:

* Test in at least one shell
* Ensure install.sh still works
* Run commands in a real Git repo

## Submitting Changes

1. Commit with a clear message
2. Push to your fork
3. Open a Pull Request

Thanks for contributing!
